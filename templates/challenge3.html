📋 Scenario

🚨 Your pipeline just broke!

We intentionally introduced a vulnerable base image (old OpenJDK / Debian with known CVEs). The security scanner (Trivy/Aqua) in the pipeline detected critical vulnerabilities and blocked the build.

BUILD FAILED — Critical vulnerability found in base image openjdk:11-jdk
CVE-2024-XXXX (CRITICAL): Remote code execution in glibc

Result: No image pushed. Deployment blocked.

📝 Steps to Complete

1

Read Pipeline Logs

Go to Jenkins ↗ and check the build logs. Look for the security scan output.

2

Identify the Vulnerability

The scanner report will show:

CRITICAL: CVE-2024-XXXX - glibc (Debian 11)
Image: openjdk:11-jdk
Fix: Update to approved base image

Approved images:
  ✅ eclipse-temurin:17-jre-alpine
  ✅ eclipse-temurin:21-jre-alpine
3

Update the Dockerfile

Change the base image in your Dockerfile to an approved, secure image:

FROM openjdk:11-jdk
FROM eclipse-temurin:17-jre-alpine
4

Rebuild & Verify

Commit the fix, push, and watch the pipeline pass. Check JFrog ↗ to verify the new image is pushed.

🎯 What You Learn

🖼️

Base Image Hygiene

Most vulnerabilities come from outdated base images, not your code.

🛡️

Security as Part of CI

Security scanning should be a mandatory gate in every pipeline.

🚫

Pipeline Gating

Pipelines should FAIL on critical vulnerabilities — this prevents insecure code from reaching production.
